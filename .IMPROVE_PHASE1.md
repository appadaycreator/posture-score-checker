# フェーズ1: 改善点提案 - posture-score-checker (2026-06-10 更新版)

## 提案サマリー
姿勢スコア診断ツールは v1.4.1 で基本機能（グラフ、履歴、共有）は完成しています。ユーザー価値と収益を最大化するための改善点を優先度順に提示します。

## P1: 結果ページの収益CTAボタン配置（M1・M5 統合実装）

**目的**: ユーザーが価値を感じた瞬間（診断完了直後）に、結果に関連するアフィリエイト商品CTAを自然な流れで配置して収益化

**問題**: 
- 結果ページに「スコアが低い → ヨガスタジオを推薦」などのアフィリエイトCTAがない
- 現在のアフィリエイトリンク（7件のA8.net + Amazon）は footer に集中しており、スクロール必須
- M1「診断/計算後の文脈CTAボタン」と M5「結果に連動したアフィリエイト推薦」が未実装

**仕様**:
- 結果ページの「スコア表示」直下に、スコアに応じた推薦商品 CTA を 2-3 件表示
- 例：
  - スコア ≤ 50: 「ヨガスタジオLAVA」「ゴルフスイング矯正」推薦
  - スコア 51-75: 「腰痛改善クッション」推薦  
  - スコア ≥ 76: 「姿勢保持補助グッズ」推薦
- 既存の 7 件の A8.net リンク（px.a8.net）は全て保持・削除禁止
- CSS でスタイリング：背景色 #FFF7ED、枠線 #fed7aa、border-radius 10px
- モバイルで 1 列表示、PC で 2-3 列表示

**実装タスク**:
- JavaScript で `displayAffiliateRecommendations(score)` 関数を新規作成
- スコア値に基づいて A8.net の既存リンク から推薦商品を動的に選別・表示
- 結果ページ（`#r-score-wrap` の直下）に新規 div `#r-affiliate-cta` を追加
- HTML の既存 A8.net リンク（行789-795）をデータセット化してメモリに保持

**DoD**:
- [ ] スコア 40・60・80 でそれぞれ異なる推薦商品が表示される
- [ ] A8.net 7 件のリンク・ビーコン img タグが全て保持されている（削除されていない）
- [ ] モバイル・PC で視認性が高く、CTR が上がっている状態
- [ ] 結果ページのスクロール量が削減されている（従来より最上部で推薦商品が見える）

## P1: 診断フロー中のガイダンス充実（M8 実装）

**目的**: 各質問の意味や選択肢の基準を明確にして、ユーザーの迷い・離脱を減らす

**問題**:
- 質問文のみで選択肢の定義が曖昧（例：「猫背ぎみ」「少し曲がる」の判断基準が不明）
- ツールチップ・? アイコンがない
- 選択肢ボタンに説明テキストがない

**仕様**:
- 各質問の右側に「?」アイコンをツールチップボタンとして配置
- ツールチップクリック時に、選択肢の判断基準をポップオーバーで表示
- 例：
  - Q1「正面から見た時の姿勢」→ ツールチップ「肩の高さ・背骨の軸が真っ直ぐか確認」
  - Q2「首・肩周辺の硬さ」→ ツールチップ「首を回せる範囲・肩が上がりやすいか」

**実装タスク**:
- 質問テンプレート内に「説明」フィールドを追加（example: `{ text: "...", desc: "..." }`)
- HTML で各質問横に `<button class="tooltip-btn">?</button>` を追加
- CSS で tooltip スタイル（background: #FFF7ED, border: 1px solid #fed7aa）
- JavaScript で `showTooltip()` 関数でポップオーバー表示

**DoD**:
- [ ] すべての質問（10問）に説明テキストが紐付けられている
- [ ] 「?」ボタンクリックで説明が表示され、再クリックで閉じる
- [ ] モバイル・PC で視認性が確保されている

## P1: 前回結果との差分比較表示（M6 部分実装）

**目的**: ユーザーが以前の診断結果と比較して進捗を実感できるようにし、再診断を促進

**問題**:
- localStorage に最大 10 件の履歴が保存されているが、前回比較表示がない
- 「進捗を見たい」というユーザーニーズに応答できていない
- M6「目標設定・達成度トラッキング」は部分実装

**仕様**:
- 結果ページに「📈 前回比較」セクションを追加
- 前回スコア vs 今回スコア を表示（例：「前回 45点 → 今回 62点 (+17点)」）
- スコア変動を矢印・色分けで表現（↑ 緑、↓ 赤、→ グレー）
- 過去 3 回のスコア推移をミニグラフで表示（既存の Chart.js グラフとは別）

**実装タスク**:
- `getHistory()` で履歴を取得し、前回・前々回のスコアを抽出
- 「前回との差分」計算ロジックを追加
- HTML で `#r-comparison` div を追加し、差分表示とミニグラフを埋め込む
- Chart.js で mini bar chart（過去 3 回）を別途レンダリング

**DoD**:
- [ ] 初回診断時は「比較」セクションが表示されない（graceful）
- [ ] 2 回目以降、「前回比較」が表示される
- [ ] スコア増減が正確に計算され、矢印・色分けで表現されている

## P2: 結果ページのモバイル最適化・スクロール削減

**目的**: モバイルユーザーの結果ページ上部への視認性向上・CTA クリック率向上

**問題**:
- HTML ファイルが長く（2000行以上）、結果ページもコンテンツが多い
- モバイル表示時、スコア表示→アフィリエイトCTA までスクロール量が多い
- 「別の条件で比較診断」ボタンが下部にあるため、再診断ハードルが高い

**仕様**:
- 結果ページの レイアウト をモバイル最適化
  - スコア表示、推薦商品CTA、簡易グラフ を「above the fold」(初期表示エリア)に配置
  - 詳細グラフ・履歴・エクサイズ は 「below the fold」に配置
- 「別の条件で比較診断」ボタンを結果ページトップに固定配置（スティッキー）
- CSS media query で mobile (max-width: 480px) の場合のレイアウト調整

**実装タスク**:
- `#result` 内の HTML 構造を再整理（順序を変更）
- モバイルでの grid/flex layout を調整
- sticky position ボタンを CSS で実装

**DoD**:
- [ ] iPhone 375px幅でスコア・CTA・ボタンが1スクロール以内に見える
- [ ] 「別の条件で比較診断」ボタンが sticky で常に操作可能
- [ ] PC でもレイアウトが崩れていない

## P2: エクサイズ提案タイミングの改善（UX）

**目的**: エクサイズ提案の重要性と実施ハードルを低くする

**問題**:
- 「おすすめ改善エクサイズ」が結果ページ中盤にあり、スクロール必須
- エクサイズの実施ハードルが高く見える（7-10 種類表示される場合がある）
- 優先度「高・中・低」のバッジは あるが、「実施期間」の目安が不明確

**仕様**:
- 「おすすめ改善エクサイズ」を「スコア直下」に移動（結果ページトップ化）
- 表示上限を「優先度が高いもの 3-5 件のみ」に厳選
- 各エクサイズに「推奨頻度」を明記（例：「毎日 5 分」「週 3 回」）
- エクサイズ実施にチェックボックスを付けて、セルフ達成度追跡を可能に

**実装タスク**:
- `getSuggestedExercises(score)` の上限を 5 件に制限
- HTML で exercise card の「実施期間」セクションを強調
- チェックボックス機能を localStorage に保存

**DoD**:
- [ ] エクサイズが 3-5 件に絞られている
- [ ] 「毎日」「週〇回」など推奨頻度が明記されている
- [ ] チェックボックスの状態が localStorage に保存される

## P2: グラフ表示のカスタマイズ（M4 強化）

**目的**: グラフの視認性向上と情報量の最適化

**問題**:
- Chart.js グラフのデフォルトスタイルがコンテンツに統一されていない
- グラフのタイトル・凡例が小さく、モバイルで見づらい
- スコア推移グラフがデフォルトのみで、カスタマイズ不足

**仕様**:
- グラフの背景色・枠線を service color (#EA580C, #FFF7ED) に統一
- モバイルでのグラフサイズを max-width: 100% で制御
- グラフタイトルを大きく・太く表示（font-size: 16px, font-weight: 700）
- スコア推移グラフに「目標ライン」（例：60点）を表示

**実装タスク**:
- Chart.js options に background color, borderColor を指定
- responsive: true, maintainAspectRatio オプションを調整
- グラフレイアウト用の container を CSS で制御

**DoD**:
- [ ] グラフが service color で統一されている
- [ ] モバイル・PC で見やすく表示されている
- [ ] 「目標ライン」が表示される（存在する場合）
